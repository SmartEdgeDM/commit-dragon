# 🐉 Contributing to Commit Dragon

First off — thanks for taking the time to contribute! Every commit helps the dragon grow. 🐲

---

## 🛠 How to Contribute

### Reporting Bugs
If you find a bug, please open an [Issue](https://github.com/SmartEdgeDM/commit-dragon/issues) using the bug report template. Include:
- What you expected to happen
- What actually happened
- Your browser and OS
- Steps to reproduce

### Suggesting Features
Have an idea? Open a [Discussion](https://github.com/SmartEdgeDM/commit-dragon/discussions) under the **Ideas** category. We love hearing what people want!

### Submitting a Pull Request
1. Fork the repo
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes to `index.html`
4. Test it locally by opening `index.html` in your browser
5. Commit your changes: `git commit -m "Add your feature"`
6. Push to your branch: `git push origin feature/your-feature-name`
7. Open a Pull Request against the `main` branch

---

## 📐 Code Style

- Everything lives in a single `index.html` file — keep it that way
- No external libraries or dependencies
- Use CSS variables for colors — don't hardcode hex values
- Keep JavaScript functions small and named clearly
- Test in Chrome, Firefox, and Safari before submitting

---

## 💡 Good First Contributions

Not sure where to start? Here are some ideas:

- Add a new dragon evolution stage
- Add a new mood state
- Add a new 8-bit sound
- Improve mobile layout
- Fix a typo in the README
- Add a new button interaction

---

## 📋 Updating the Changelog

Every contribution should include an update to `CHANGELOG.md`. Add your change under a new version number at the top of the file following this format:

```
## [X.X.X] — YYYY-MM-DD

### Added
- Description of what you added

### Fixed
- Description of what you fixed
```
